ARCHITECTURAL SPECIFICATION DOCUMENTMODULE: node_input_guard (Security & Memory Enrichment Intake)I. BUSINESS OVERVIEW & INTENTnode_input_guard đóng vai trò là cửa ngõ an toàn đầu tiên (Intake & Security Gate) của toàn bộ đồ thị Multi-Agent. Nó xử lý hai nhiệm vụ cốt lõi trước khi bàn giao quyền điều khiển cho node_supervisor:Safety Guarding: Thực hiện phân tích nội độc hại, Prompt Injection, rò rỉ dữ liệu hoặc các từ khóa vi phạm chính sách hệ thống (System Policies).Deterministic Memory Enrichment: Tự động hóa việc tra cứu và nạp dữ liệu từ kho lưu trữ dài hạn (Long-term Store bao gồm User Profile và các Episodic Memory tương đồng nhất) vào Bus nội bộ thông qua Code thuần (Deterministic), tối ưu hóa tuyệt đối chi phí Token và triệt tiêu độ trễ (Latency) của LLM trước khi Router xử lý.II. COMPONENT FILES SPECIFICATION1. Core Contract (input_guard_protocol.py)Định nghĩa giao ước dữ liệu bất biến (Immutable Data Contracts) đóng vai trò làm kết quả đầu ra của tầng nghiệp vụ thuần.Tiêu chuẩn kỹ thuật: Sử dụng Pydantic v2, cấu hình ConfigDict(frozen=True, extra="ignore").Cấu trúc Object: InputGuardResultis_safe (bool): Trạng thái an toàn của Request.risk_score (float): Điểm số rủi ro phát hiện ($0.0 \rightarrow 1.0$).security_feedback (str): Lý do chi tiết nếu phát hiện vi phạm chính sách bảo mật.user_profile (dict): Bản ghi hồ sơ khách hàng được kéo từ Store (nếu an toàn).relevant_episodes (list[str]): Danh sách chuỗi tóm tắt các trải nghiệm tương đồng trong quá khứ được truy xuất từ Vector Store (nếu an toàn).2. Pure Domain Execution (input_guard_service.py)Chứa đựng toàn bộ logic nghiệp vụ thuần túy (Deterministic Logic & I/O Calls). Nhận tham số thô (str, dict), độc lập hoàn toàn với State của LangGraph.Tiêu chuẩn kỹ thuật: class InputGuardService với hàm xử lý bất đồng bộ async def execute(self, user_text: str, user_id: str, embed_model: Any) -> InputGuardResult.Logic thực thi nội bộ:Quét nội dung user_text dựa trên bộ quy tắc bảo mật hệ thống (Blacklist Keywords / Injection Patterns).Nếu vi phạm: Lập tức trả về InputGuardResult với is_safe=False kèm feedback lỗi, bỏ qua bước đọc Store.Nếu an toàn: Gọi dịch vụ I/O chọc thẳng vào DB/Cache bằng user_id để lấy Profile, và sử dụng embed_model được truyền vào từ Container để làm phép so sánh Vector (Semantic Search) tìm kiếm các relevant_episodes phù hợp. Đóng gói toàn bộ vào Object Pydantic đóng băng và trả về.3. Mainboard Gatekeeper (adapter_input_guard.py)Điểm đấu nối trực tiếp vào đồ thị LangGraph, đóng vai trò chuyển dịch dữ liệu giữa mạng Bus hệ thống (MainBus) và tầng Service nghiệp vụ.Tiêu chuẩn kỹ thuật: Hàm xử lý đồ thị async def node_input_guard(state: MainBus, config: RunnableConfig = None) -> dict.Tuân thủ Safe Post-Guard (Chống sập Đồ thị): Vì đây là Node đầu vào (START), nó không có Node liền trước. Do đó, bước Safe Post-Guard sẽ chuyển sang kiểm tra tính toàn vẹn của gói tin khởi tạo ban đầu trên MainBus. Nếu gói tin thiếu user_id hoặc cấu trúc tin nhắn thô trống, lập tức hạ cánh an toàn bằng cách phát StandardFrame.emit với trạng thái "FAILED" và ghi lỗi vào trường error. Không dùng câu lệnh raise.Dependency Injection & Context Extraction: Gọi Container thông qua await get_ctx() để lấy thực thể default_embed, bóc tách tin nhắn mới nhất từ mảng tin nhắn của Bus.BodyFrame Schema Mapping (Strict Constraint): Đồng bộ hóa kết quả từ Service sang đúng định dạng 8 trường cố định của BodyFrame:status: "SUCCESS" nếu an toàn, hoặc "FAILED" nếu vi phạm bảo mật / lỗi hệ thống.text: Nội dung sạch của User (UI Single Source of Truth).records: Chứa danh sách các relevant_episodes tìm được.entities: Không xử lý (để trống).state: Lưu trạng thái runtime {"is_safe": True, "process_completed": True}.metrics: Lưu thông số hệ thống {"risk_score": ...}.context: Lưu thông tin user_profile và thông tin debug liên quan.error: Chuỗi thông báo lỗi chi tiết nếu status == "FAILED".III.
